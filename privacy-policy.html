Privacy Policy

Last updated: April 11, 2024

Introduction

This Privacy Policy explains how Auto-Journal ("we", "our", or "us") collects, uses, and protects your information when you use our browser extension. We are committed to ensuring that your privacy is protected and that you have a transparent understanding of our data practices.

Auto-Journal is designed with privacy as a core principle. The extension primarily stores data locally on your device, with optional backend processing for AI summarization.

Required Permissions

Auto-Journal requires the following browser permissions to function properly:

storage

This permission allows the extension to save your journal entries, highlights, and settings locally on your device. Without this permission, the extension would not be able to remember your browsing history or preferences.

tabs

This permission allows the extension to detect when you navigate to a new page and access information about the current tab (URL, title, favicon). This is essential for logging your browsing activity.

activeTab

This permission allows the extension to interact with the currently active tab, including extracting content for summarization and enabling the highlighting feature. It is only active when you explicitly interact with the extension.

scripting

This permission allows the extension to inject content scripts into web pages to extract content and enable highlighting functionality. It is necessary for the core features of the extension to work.

contextMenus

This permission allows the extension to add items to the browser's context menu (right-click menu), enabling the "Save to Auto-Journal" feature for highlighted text.

downloads

This permission allows the extension to export your journal data as a JSON file when you use the export feature. It is only used when you explicitly choose to export your data.

host_permissions

The extension requests permission to communicate with our backend API (https://auto-journal-browser-extension.onrender.com/*) for AI summarization and optional cloud storage. This is only used when these features are enabled.

Information We Collect

Auto-Journal collects the following information:

  • Browsing activity: URLs, page titles, visit timestamps, and favicons of pages you visit
  • Page content: A portion of the visible text content from visited pages (limited to the first 1000 words)
  • Screenshots: Optional screenshots of visited pages (if enabled in settings)
  • Highlights: Text that you explicitly select and save using the highlighting feature
  • User preferences: Settings such as blacklisted domains and feature toggles

What we DO NOT collect:

  • Passwords, form data, or login credentials
  • Private browsing history (when using incognito/private mode)
  • Content from secure pages (HTTPS pages with login forms)
  • Personal messages or emails
  • Financial information
  • Any data from domains you've blacklisted in the settings

How We Use Your Information

The information we collect is used for the following purposes:

  • Creating and maintaining your personal browsing journal
  • Generating AI summaries of visited pages
  • Providing search and filtering functionality
  • Auto-tagging and categorizing content
  • Improving the extension's functionality and user experience

We do not sell, rent, or share your data with third parties for marketing or advertising purposes. Your data is primarily stored locally on your device, with optional processing on our secure backend for AI summarization.

Data Storage & Security

Auto-Journal stores your data in the following ways:

  • Local storage: By default, all journal entries, highlights, and settings are stored locally on your device using the browser's storage API. This data never leaves your device unless you explicitly enable cloud syncing.
  • Backend processing: When AI summarization is enabled, page content is temporarily sent to our secure backend for processing. This content is not permanently stored on our servers unless you enable cloud syncing.
  • Cloud storage (optional): If you choose to enable cloud syncing (a future feature), your journal data will be stored in our MongoDB database with encryption and secure authentication.

We implement appropriate security measures to protect your data, including:

  • HTTPS encryption for all API communications
  • Secure authentication for cloud syncing
  • Regular security audits and updates
  • Data minimization practices

Third-Party Services

Auto-Journal uses the following third-party services:

  • Gemini 2.0 Flash Lite API: Used for AI summarization of page content. When this feature is enabled, page content is sent to Google's Gemini API for processing. Please refer to Google's AI Privacy Policy for more information.
  • MongoDB Atlas: Used for cloud storage when syncing is enabled. Please refer to MongoDB's Privacy Policy for more information.
  • Render: Hosting platform for our backend API. Please refer to Render's Privacy Policy for more information.

User Rights

As a user of Auto-Journal, you have the following rights regarding your data:

  • Access: You can view all your journal entries and highlights through the extension's interface.
  • Export: You can export all your data as a JSON file using the export feature in the settings.
  • Delete: You can delete individual entries or clear all data using the options in the extension.
  • Control: You can control what data is collected by adjusting the settings, including blacklisting domains and toggling features.

Changes to This Privacy Policy

We may update this Privacy Policy from time to time to reflect changes in our practices or for other operational, legal, or regulatory reasons. When we make changes, we will update the "Last updated" date at the top of this policy.

Significant changes will be notified through the extension interface or via email if you have provided one. We encourage you to review this Privacy Policy periodically to stay informed about our data practices.

Contact Information

If you have any questions, concerns, or requests regarding this Privacy Policy or our data practices, please contact us by:

  • Opening an issue on our GitHub repository
  • Contacting the developer directly through GitHub
